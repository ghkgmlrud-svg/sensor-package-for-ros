# MW-AHRS-X1 ROS2 Package

ROS2 package for receiving MW-AHRS-X1 serial IMU data and publishing ROS topics.

## Package and Node

| Item | Name |
| --- | --- |
| ROS2 package | `mw_ahrs_x1_pkg` |
| Node executable | `mw_ahrs_x1_node` |
| Python node file | `mw_ahrs_x1_pkg/mw_ahrs_x1_node.py` |

## Startup Behavior

When the node starts, it sends these commands to the sensor:

```text
+++
ss=4
```

This matches the Python serial test flow used before creating the ROS2 node.

## Run

```bash
ros2 run mw_ahrs_x1_pkg mw_ahrs_x1_node
```

Run with custom parameters:

```bash
ros2 run mw_ahrs_x1_pkg mw_ahrs_x1_node --ros-args \
  -p port:=/dev/ttyUSB0 \
  -p baudrate:=115200 \
  -p output_hz:=5.0
```

## Parameters

| Parameter | Default | Description |
| --- | --- | --- |
| `port` | `/dev/ttyUSB0` | Serial device path |
| `baudrate` | `115200` | Serial baudrate |
| `timeout` | `1.0` | Serial read timeout in seconds |
| `output_hz` | `50.0` | Timer rate used by the ROS node |
| `frame_id` | `imu_link` | Frame ID in `/imu` header |
| `enter_command_mode` | `true` | Send command mode string before startup command |
| `command_mode_string` | `+++` | Command mode string |
| `command_mode_delay` | `1.5` | Delay after command mode string |
| `startup_command` | `ss=4` | Startup command sent to sensor |
| `startup_command_delay` | `0.5` | Delay after startup command |

## Topics

| Topic | Type | Description |
| --- | --- | --- |
| `/imu/raw` | `std_msgs/msg/String` | Raw serial line from the sensor |
| `/imu/data` | `std_msgs/msg/Float64MultiArray` | Numeric values parsed from raw text |
| `/imu` | `sensor_msgs/msg/Imu` | Standard ROS IMU message |

## IMU Message Mapping

`sensor_msgs/msg/Imu` does not contain direct `roll`, `pitch`, and `yaw` fields. The node converts the first three numeric values into quaternion orientation.

Current assumption:

```text
values[0] = roll  in degrees
values[1] = pitch in degrees
values[2] = yaw   in degrees
values[3:6] = linear acceleration, if available
values[6:9] = angular velocity in degrees/sec, if available
```

Publish behavior:

```text
3+ values -> /imu orientation
6+ values -> /imu orientation + linear_acceleration
9+ values -> /imu orientation + linear_acceleration + angular_velocity
```

If the real sensor output order is different, update the mapping in `mw_ahrs_x1_pkg/mw_ahrs_x1_node.py`.

## Timestamp

The `/imu` timestamp is generated by the ROS2 node when it creates the message:

```python
msg.header.stamp = self.get_clock().now().to_msg()
```

With normal settings, this is system time. If `use_sim_time` is enabled, it follows `/clock` simulation time.
